OVERVIEW

This was my favorite project so far. We first created a cloth composed of point masses connected by different types of springs and simulated the forces using verlet integration. Then we implemented collision handling with other objects such as spheres and planes as well as collisions between point masses. Finally, we implemented vertex and fragment shaders which allowed us to apply lighting and textures to the cloth in real time. This project helped me gain a deeper understanding of how physics is simulated in real world applications as well as how shaders can take advantage of GPUs to quickly compute pixel colors.

Part 5

Explain in your own words what is a shader program and how vertex and fragment shaders work together to create lighting and material effects.

In our previous projects, just 1 frame of a scene required minutes of rendering time. This is simply not acceptable in real world applications. This is where our shader program comes in which takes advantage of parallelized computations on the GPU and allows us to render our scene in real time. Vertex shaders allow us to augment the positions and normals of vertices and pass on geometric values to the fragment shaders which calculate a RGBA value for each fragment (which is basically a pixel or more specifically a pixel sample).

Explain the Blinn-Phong shading model in your own words. Show a screenshot of your Blinn-Phong shader outputting only the ambient component, a screen shot only outputting the diffuse component, a screen shot only outputting the specular component, and one using the entire Blinn-Phong model.

The Blinn-Phong shading model is an easy to implement estimate of the lighting. We compute a half-way vector between the vector from the camera to the point and from the point to the light source. The closer the half-way vector is to the surface normal, the brighter the point is.
